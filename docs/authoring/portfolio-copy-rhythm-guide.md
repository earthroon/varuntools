# Portfolio copy rhythm guide

Commit 126 기준 문서다. 이 가이드는 featured portfolio page의 카피와 editorial block 사용 밀도를 정리한다.

## Purpose

Portfolio copy should not stop at listing what was made. It should show why the work exists, what pressure started it, how the structure holds it, and which technical layer makes it work.

## Heading rhythm

### Major heading

Use a major editorial heading when a section needs an entrance, not just a label.

Good major headings:

- open the scene
- tell the reader how to read the page
- connect abstract intent to real work axes

Avoid using major headings for every large paragraph. If everything is major, nothing has weight.

### Middle heading

Use a middle editorial heading for a structural hinge. It should announce the role of the next paragraph or column group.

Good middle headings:

- separate problem, solution, implementation, and result
- make the next block easier to scan
- avoid becoming pure mood text

### Minor heading

Use a minor editorial heading inside a card, column, or small judgment unit. It should be short and stable.

Good minor headings:

- name a specific function or judgment
- avoid long subtitles
- keep the column rhythm compact

## Column rhythm

### Problem / solution pattern

Use a 2-column block when the reader needs comparison.

The two sides should carry similar weight. One side should not be a poetic fragment while the other becomes a technical dump.

### Emotion / structure / technology pattern

Use a 3-column block to show the VARUNTOOLS thinking stack.

- Emotion: the pressure, lack, friction, or felt problem
- Structure: the state owner, SSOT, content flow, or information shape
- Technology: the actual implementation, validation, renderer, worker, or build artifact

This pattern should make the work easier to understand. It should not become a decorative slogan.

## Overuse rule

Editorial blocks are not a replacement for Markdown.

Use existing Markdown headings for ordinary document flow. Use editorial-title and editorial-columns only where visual hierarchy or comparison improves comprehension.

## Copy rules

1. Abstract words must connect to an operating structure.
2. Technical stacks should not be listed without purpose.
3. Show what was structured before saying what was built.
4. Emotional language is strongest when it is tied to a function.
5. Column sentences should carry similar weight.
6. Major headings open a scene, middle headings fold structure, minor headings pin a judgment.
