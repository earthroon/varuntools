# Easy Markdown Wrapper Roadmap

> Easy Markdown is an authoring wrapper for VarunTools.  
> It does not replace the existing canonical Markdown directives.  
> It gives authors a smaller `@command` surface and compiles that source into the existing `index.md` format.

## 1. Purpose

Easy Markdown exists to lower the writing cost of VarunTools pages without weakening the current renderer.

```txt
Author-friendly source
  work.easy.md / page.easy.md

Canonical generated output
  index.md

Existing runtime
  markdown-it + VarunTools directives + Vue renderer
```

The wrapper lets an author write:

```md
@problem
The page creation flow was too manual.
@end
```

and compile it into the canonical directive form used by the site.

## 2. Current authoring model

| Mode | Source of truth | Generated file | Primary use |
|---|---|---|---|
| Manual Markdown | `index.md` | none | narrative pages, case studies, hand-authored Works |
| CSV-authored pages | `page.csv` | `index.md` | repeated blocks, catalog-like pages, structured content |
| Easy Markdown | `work.easy.md` or `page.easy.md` | `index.md` | author-friendly Works/case-study/page writing |

## 3. SSOT rules

```txt
Manual Markdown page:
  index.md is the SSOT.

CSV-authored page:
  page.csv is the SSOT.
  index.md is generated.

Easy Markdown page:
  work.easy.md or page.easy.md is the SSOT.
  index.md is generated.
```

Hard rules:

```txt
- Do not silently convert manual index.md pages into generated pages.
- Do not allow page.csv and *.easy.md in the same page folder.
- Do not delete or replace existing ::directive syntax.
- Do not edit generated index.md directly.
- Generated index.md must include a GENERATED marker.
```

Recommended generated marker:

```md
<!--
GENERATED FROM work.easy.md.
Do not edit this file directly.
Run: npm run easy:compile
-->
```

## 4. Target pipeline

```txt
work.easy.md / page.easy.md
→ scripts/compile-easy-markdown.mjs
→ index.md
→ npm run validate:content
→ Vue Markdown renderer
→ Vite build
```

The compiler should be deterministic. A `--check` mode should fail when generated `index.md` is stale.

## 5. Initial scope

The first implementation should focus on Works and case-study pages.

Phase 1 should cover:

```txt
# Title
@description
@slug
@public / @hidden / @draft
@cover
@thumb
@work
@role
@stack
@tools
@tags
@summary
@problem
@decision
@solution
@process
@result
```

Store/product wrappers should remain documented but deferred until the portfolio authoring flow is stable.

## 8.5 Scaffolding integration

Status: implemented in commit 10.

Easy Markdown pages can now be scaffolded from templates.

```powershell
npm run new:work:easy -- varuntools --title "VARUNTOOLS"
npm run new:page:easy -- guide markdown-guide --title "Markdown Guide"
```

Scaffolded pages create the Easy source, generated `index.md`, local README, placeholder assets, and asset folders. Existing folders are not overwritten unless `--force` is explicitly provided.

## 11. Non-goals

```txt
- Replacing canonical ::directive syntax
- Removing CSV authoring
- Making generated index.md the editing surface
- Autocorrecting unknown commands silently
- Creating hidden pages without explicit status/visibility commands
```
