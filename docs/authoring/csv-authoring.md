# CSV Authoring

## Rule

`page.csv` is the authoring source. `index.md` is generated output.

Do not edit generated `index.md` directly for CSV-based pages. Edit `page.csv` and regenerate the Markdown.

## Commands

```bash
npm run csv:page -- src/content/pages/works/project-name/page.csv
npm run csv:pages
npm run new:page -- works project-name --csv
```

## Columns

```csv
block,title,body,src,alt,caption,thumb,layout,kind,options,meta
```

| Column | Purpose |
|---|---|
| `block` | Block type to generate |
| `title` | Page title, heading, box title, video title, gallery title |
| `body` | Description, paragraph body, box body, gallery caption |
| `src` | Image/video source, or before image for before-after |
| `alt` | Image alt text |
| `caption` | Caption text |
| `thumb` | Thumbnail or poster path |
| `layout` | Gallery layout such as `strip`, `grid`, `compact` |
| `kind` | Heading level or markdown-box type |
| `options` | `key=value; key=value` options |
| `meta` | Gallery item metadata |

## Blocks

- `page`

- `raw`

## Options

Options use semicolon-separated key/value pairs.

```txt
tags=work|design; order=10; gallery.autoMini=true
```

`|` creates an array. `true` and `false` become booleans. Numbers become numbers.

## Product block

Use `product` after the `page` block for product pages.

```csv
product,Product Title,상품 판매 정보입니다.,./images/cover.webp,,,./images/cover.webp,,,type=physical; sku=VT-PRODUCT-001; price=12000; currency=KRW; priceVisible=true; status=coming-soon; checkoutProvider=toss-payments; checkoutUrl=; downloadProvider=cloudflare; downloadUrl=; showWhenUnavailable=true,
```

`checkoutUrl` is for Toss Payments links. `downloadUrl` is reserved for Cloudflare digital delivery.

## Store Metadata v2

Product CSV files can carry store-facing metadata without hand-editing generated Markdown.

Page-level options commonly used by store/catalog pages:

| Option | Purpose |
|---|---|
| `status=active` | Public page status. Do not use `published`. |
| `visibility=public` | Show the page in generated collections. |
| `featured=true` | Allows homepage featured sections to pick the page. |
| `series=` | Brand or creative series grouping. |
| `collection=` | Store collection grouping. |
| `releaseDate=` | Optional launch/publication date in `YYYY-MM-DD`. |
| `externalUrl=` | Optional external reference URL. |
| `license=` | Optional usage/license label. |
| `role=` / `client=` | Work/service metadata when needed. |

Product-level options accepted by the `product` block:

| Option | Purpose |
|---|---|
| `type=` | `physical`, `digital`, `service`, or `external`. |
| `status=` | `draft`, `coming-soon`, `available`, `sold-out`, or `hidden`. |
| `sku=` | Store SKU or edit placeholder. |
| `price=` / `currency=` / `priceVisible=` | Display price contract. Empty `price=` is allowed for inquiry services. |
| `checkoutUrl=` | Toss Payments or direct checkout link. Do not fake this. |
| `externalStoreUrl=` | External store link when checkout is hosted elsewhere. |
| `inquiryUrl=` | Inquiry/contact link for service or quote-based products. |
| `downloadProvider=` / `downloadUrl=` | Digital delivery metadata. |
| `license=` | Usage/license label for digital files, presets, templates, or code. |
| `series=` / `collection=` | Store grouping metadata. |
| `material=` / `size=` | Physical product detail metadata. |
| `shippingNote=` / `refundNote=` | Product-specific trust block override. |
| `digitalDeliveryNote=` | Digital product delivery override. |
| `policyNote=` | Store-guide note shown around product trust blocks. |
| `externalUrl=` | External reference/detail link, separate from checkout. |

Generated product pages should keep both product directives:

```md
::product-cta
::

::product-trust
::
```

## Commit 87: Block schema and diagnostics

CSV authoring now has an explicit block contract in `scripts/lib/csv-block-schema.mjs` and diagnostic formatting in `scripts/lib/csv-diagnostics.mjs`.

### Diagnostics model

Diagnostics include level, code, row number, block, field, option key, and an optional hint.

```txt
[error] [CSV_REQUIRED_FIELD_MISSING] row 7, block "image", field "src": image block requires "src".
[warning] [CSV_UNKNOWN_OPTION] row 12, block "image", option "foo": image block does not define option "foo".
```

### Required versus recommended

- Missing required fields fail generation.
- Missing recommended fields warn but do not block generation.
- Unknown options warn in loose mode and fail in strict mode.

### Strict mode

```bash
npm run csv:page -- src/content/pages/products/dummy-catalog/page.csv --strict
```

Strict mode is intended for fixture and release checks. Keep normal authoring in loose mode while migrating old CSV files.

### Report mode

```bash
npm run csv:page -- src/content/pages/products/dummy-catalog/page.csv --report --dry-run
```

Report mode prints row count, block counts, and diagnostic totals. It is useful before committing generated Markdown.

### Gallery structure rules

- `gallery-item` must be between `gallery-start` and `gallery-end`.
- Nested `gallery-start` is invalid.
- `gallery-end` without an open gallery is invalid.
- An unclosed `gallery-start` fails generation.

### Raw block caution

`raw` is still available for repository-authored content, but it emits a warning because it bypasses the safer block contract. Do not enable `raw` for external user-provided CSV input.

### Validation commands

```bash
npm run smoke:csv-authoring
npm run smoke:csv-diagnostics
npm run validate:content
```

## Commit 88: Options v2

The `options` cell now supports legacy options and a stricter typed authoring syntax.

### Legacy syntax remains valid

```txt
tags=work|design; order=10; featured=true
```

- `|` creates a legacy array.
- `true` and `false` become booleans.
- Numeric values become numbers.

### Quoted strings

Use double quotes when a value contains `;`, `|`, `=`, or prose punctuation that should remain part of the value.

```txt
summary="작업은 느리게; 그러나 정확하게 움직였다"; quote="구조|리듬=감각"
```

### Escaped delimiters

Use a backslash when you want a delimiter in an unquoted value.

```txt
summary=작업은 느리게\; 그러나 정확하게; label=UI\|UX; equation=a\=b
```

Supported escapes are:

```txt
\; \| \= \" \\ \, \[ \]
```

### Bracket arrays

Prefer bracket arrays for new CSV authoring when values may contain spaces or punctuation.

```txt
tags=[work, design, tool]; stack=[Vue, TypeScript, Cloudflare Workers]
labels=["UI/UX", "CSV;Markdown"]
```

Array values are typed, so numbers, booleans, and `null` are preserved.

```txt
weights=[1, 2, 3]; flags=[true, false, null]
```

### Null values

```txt
client=null
```

### Nested keys

Dot notation still creates nested objects.

```txt
mood.tone=almond-paper; mood.density=high; gallery.autoMini=true
```

Nested option keys are now validated against the block schema. If a schema allows `mood`, then `mood.*` is allowed. If it allows only `mood.tone`, only that leaf key is allowed.

### Syntax failures

Invalid option syntax fails in both loose and strict mode. Unknown options still warn in loose mode and fail in strict mode.

Examples that fail:

```txt
summary="unclosed quote
tags=[work, design
=missingKey
```

Run:

```bash
npm run smoke:csv-options
npm run smoke:csv-diagnostics
npm run smoke:csv-authoring
```

## Portfolio case-study blocks

Commit 89 adds portfolio-oriented CSV blocks. These blocks are still rendered through known Markdown directives (`markdown-box` and `gallery-strip`) so content validation does not need new Vue directive wiring yet.

### `portfolio-hero`

Use this as the top case-study block after the `page` row.

```csv
portfolio-hero,VarunTools Store,"보안과 콘텐츠 발행이 분리된 포트폴리오형 스토어",./cover.webp,Cover,대표 이미지,./thumb.webp,split,case-study,"role=[Design Engineer, Frontend]; stack=[Vue, TypeScript, Cloudflare Workers]; year=2026; featured=true",
```

Required: `title`, `body`.
Recommended: `src`, `alt`, `thumb`, `options.role`, `options.stack`, `options.year`.

### Case narrative blocks

Use these to show the structure of the work rather than only the final image.

```txt
problem
solution
process
decision
result
```

Each requires `body` and recommends `title`.

`decision` is rendered as a `markdown-box` with `type: decision`; `problem` uses `type: warning`; `result` uses `type: tip`; the rest use `type: note`.

### Summary and stack blocks

```txt
work-summary
role-stack
tool-stack
metric
quote
```

- `work-summary` requires `title` and `body`.
- `role-stack` requires one of `title`, `body`, `options.role`, `options.stack`, or `options.tools`.
- `tool-stack` requires one of `body`, `options.stack`, `options.tools`, `options.language`, `options.runtime`, or `options.storage`.
- `metric` requires `title` and recommends `body` or `options.value`.
- `quote` requires `body` and preserves punctuation inside quoted text.

### Case gallery

Use a case gallery when the images belong to a single case-study section.

```csv
case-gallery-start,주요 화면,,,,,,,,columns=2; variant=grid,
case-gallery-item,,관리자 감사 장부 화면,./audit-log.webp,Audit log,관리자 dry-run 기록,,,,label=audit,
case-gallery-end,,,,,,,,,,
```

Rules:

- `case-gallery-item` must appear between `case-gallery-start` and `case-gallery-end`.
- Nested `case-gallery-start` is invalid.
- An unclosed case gallery fails diagnostics.

### Related works

Use `related-works` with `options.items`.

```csv
related-works,관련 작업,,,,,,,,items=[csv-authoring, varuntools-store],
```

`related-works` requires either `body` or `options.items`.

### Validation

Run:

```bash
npm run smoke:csv-portfolio-blocks
npm run smoke:csv-options
npm run smoke:csv-diagnostics
npm run smoke:csv-authoring
```

## Portfolio frontmatter v2

Commit 90 adds a `work` object to generated frontmatter so portfolio pages can be sorted, filtered, featured, and recommended without scraping body content.

Preferred authoring uses the `work.*` namespace on the `page` row:

```csv
block,title,body,src,alt,caption,thumb,layout,kind,options,meta
page,VarunTools Store,"보안과 콘텐츠 발행이 분리된 포트폴리오형 스토어",./cover.webp,Cover,,./thumb.webp,,,"slug=works/varuntools-store; kind=work; status=active; visibility=public; work.type=case-study; work.status=published; work.featured=true; work.weight=90; work.role=[Design Engineer, Frontend]; work.stack=[Vue, TypeScript, Cloudflare Workers]; work.tags=[store, security, portfolio]; work.mood.tone=almond-paper",
```

Generated frontmatter:

```yaml
work:
  type: case-study
  status: published
  featured: true
  weight: 90
  role:
    - Design Engineer
    - Frontend
  stack:
    - Vue
    - TypeScript
    - Cloudflare Workers
  tags:
    - store
    - security
    - portfolio
  mood:
    tone: almond-paper
```

### Work metadata fields

| Field | Type | Meaning |
|---|---:|---|
| `work.type` | string | `case-study`, `tool`, `visual`, `service`, `experiment`, `store`, `system` |
| `work.status` | string | `draft`, `published`, `archived`, `private` |
| `work.featured` | boolean | Home/list highlight flag |
| `work.weight` | number | Sort priority for featured surfaces |
| `work.period` | string | Display period such as `2026` or `2025-2026` |
| `work.year` | number | Representative year |
| `work.client` | string | Client name or `internal` |
| `work.role` | string[] | Role chips |
| `work.stack` | string[] | Technology stack chips |
| `work.tools` | string[] | Tool chips |
| `work.tags` | string[] | Filter/search tags |
| `work.category` | string | High-level category |
| `work.summary` | string | Card-oriented short summary |
| `work.mood.*` | object | Portfolio tone metadata |
| `work.links.*` | object | `demo`, `repo`, `caseStudy` links |

### Fallback policy

`frontmatter.work` is generated only when one of these is true:

1. The `page` row uses `work.*` options.
2. The page is explicitly authored as a work page through `kind=work`.
3. A `portfolio-hero` block exists and provides fallback metadata.

SSOT priority:

```txt
page.options.work.* > page legacy work-like options > portfolio-hero fallback > no work object
```

This prevents product pages that happen to use `tags` or `featured` from accidentally receiving a portfolio `work` object.

## Preview / Diff / Check

Commit 91 adds observation modes for CSV-authored Markdown. These modes let you inspect generated output before writing `index.md`.

### Preview

```bash
npm run csv:page -- src/content/pages/works/project-name/page.csv --preview
```

`--preview` prints generated Markdown to stdout and does not write `index.md`.

`--dry-run` remains a compatibility alias for `--preview`.

### Diff

```bash
npm run csv:page -- src/content/pages/works/project-name/page.csv --diff
```

`--diff` compares the current `index.md` with the Markdown generated from `page.csv` and prints a small dependency-free line diff.

If the output file does not exist, the generated Markdown is compared against an empty file.

### Check

```bash
npm run csv:page -- src/content/pages/works/project-name/page.csv --check
```

`--check` is intended for CI. It exits with code `1` when the generated Markdown differs from the current `index.md`.

### Report

```bash
npm run csv:page -- src/content/pages/works/project-name/page.csv --report
```

`--report` prints source/output paths, row count, block counts, changed state, and diagnostics counts.

### Write with observation modes

Observation modes do not write files by default. To inspect and write in one command, pass `--write` explicitly:

```bash
npm run csv:page -- src/content/pages/works/project-name/page.csv --diff --report --write
```

Recommended authoring loop:

```bash
npm run csv:page -- path/to/page.csv --report --diff
npm run csv:page -- path/to/page.csv
npm run csv:page -- path/to/page.csv --check
```

## CSV Asset Guard

Commit 93 adds asset validation during CSV generation. The validator checks `src`, `thumb`, gallery items, before/after images, video posters, and portfolio hero images before Markdown is written.

Local relative paths resolve from the directory that contains the `page.csv` file. Public absolute paths resolve under `public/`.

```txt
./images/cover.webp      # page-local
/assets/cover.webp       # public/assets/cover.webp
```

Unsafe protocols such as `javascript:`, `data:`, `file:`, and `blob:` are blocked. External `https://` assets are warnings in loose mode and become errors in strict mode when the host is not allowlisted.

The guard does not fix files. It reports problems so the author can correct the source CSV or asset folder intentionally.

See `docs/authoring/csv-asset-guard.md` for the full contract.

## Commit 95 — Dedicated portfolio render components

Portfolio CSV blocks now generate dedicated Markdown directives instead of defaulting to generic `markdown-box` output.

| CSV block | Generated directive |
| --- | --- |
| `portfolio-hero` | `portfolio-hero` |
| `work-summary` | `work-summary` |
| `role-stack` | `role-stack` |
| `problem` / `solution` / `process` / `decision` / `result` | `case-section` |
| `metric` | `metric-card` |
| `tool-stack` | `tool-stack` |
| `quote` | `quote-block` |
| `case-gallery-start` / `case-gallery-item` / `case-gallery-end` | `case-gallery` |
| `related-works` | `related-works` |

The renderer keeps the existing Vue tone/manner: calm paper-like surfaces, restrained chips, existing design tokens, and responsive layouts. It does not add lightbox or animation behavior beyond the existing gallery event contract.

## Portfolio related links

Use `related-works` for internal portfolio relationships only. Accepted item formats are `slug`, `/works/slug`, and `works/slug`. External URLs are rejected from `related-works`; place external destinations in `work.links.demo`, `work.links.repo`, or another explicit link field instead.

## Commit 100 - Portfolio presets

Portfolio pages can now be scaffolded from CSV presets:

```bash
npm run new:page -- works my-case --csv --type case-study
npm run new:page -- works/my-tool --csv --type tool
```

Preset pages are generated as `work.status=draft` and `work.featured=false`. Existing `page.csv` files are never overwritten. The generated `page.csv` remains the authoring SSOT; regenerate Markdown through `npm run csv:pages`.

## EWA gallery metadata

`case-gallery-item` rows support nested `media.*` options for lightbox-only EWA processing hints:

```txt
media.ewaPreset=ui-low-ring
media.ewaMode=adaptive-tile
media.pixelSafe=true
media.ewaEnabled=false
```

These options are metadata only. They do not rewrite the source image.
