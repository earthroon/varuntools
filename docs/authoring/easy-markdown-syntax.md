# Easy Markdown Syntax Mapping

> This document maps author-friendly Easy Markdown commands to the existing VarunTools Markdown/frontmatter model.  
> The canonical renderer remains the existing `index.md` + `::directive` pipeline.

## 1. File names

| Easy source | Generated output | Use |
|---|---|---|
| `work.easy.md` | `index.md` | Works pages and portfolio case studies |
| `page.easy.md` | `index.md` | General pages |

Generated files should contain a marker:

```md
<!--
GENERATED FROM work.easy.md.
SOURCE HASH: sha256:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
Do not edit this file directly.
Run: npm run easy:compile
-->
```

## 2. Frontmatter commands

| Easy syntax | Output |
|---|---|
| `# Title` | `title` |
| `@description ...` | `description` |
| `@slug works/foo` | `slug` |
| `@public` | `visibility: public` |
| `@hidden` | `visibility: hidden` |
| `@draft` | `status: draft` |
| `@cover ./cover.webp` | `cover` |
| `@thumb ./thumb.webp` | `thumbnail` |
| `@work type=system status=published year=2026 featured=true` | `work.*` |
| `@role Designer, Developer` | `work.role[]` |
| `@stack Vue3, Vite, TypeScript` | `work.stack[]` |
| `@tools GitHub Actions, Markdown-it` | `work.tools[]` |
| `@tags portfolio, cms` | `work.tags[]` |

Example:

```md
# VARUNTOOLS

@description Vue3 + Vite based portfolio publishing system.
@slug works/varuntools
@public
@cover ./cover.svg
@thumb ./thumb.svg
@work type=system status=published year=2026 featured=true weight=1
@role Designer, Developer
@stack Vue3, Vite, TypeScript
@tools GitHub Actions, Markdown-it
@tags portfolio, markdown, cms
```

## 3. Case section commands

Status: implemented in commit 05.

| Easy command | Canonical directive | Default title |
|---|---|---|
| `@summary` | `::work-summary` | none |
| `@problem` | `::case-section type: problem` | 문제 |
| `@decision` | `::case-section type: decision` | 판단 |
| `@solution` | `::case-section type: solution` | 해결 |
| `@process` | `::case-section type: process` | 과정 |
| `@result` | `::case-section type: result` | 결과 |

### `@summary`

```md
@summary
요약을 작성합니다.
@end
```

Compiles to:

```md
::work-summary
::
요약을 작성합니다.
::
```

### `@problem` and other case sections

```md
@problem axis=workflow title="작성 병목"
본문을 작성합니다.
@end
```

Compiles to:

```md
::case-section
type: problem
title: 작성 병목
axis: workflow
::
본문을 작성합니다.
::
```

Supported optional attributes:

| Attribute | Purpose |
|---|---|
| `title=` | Override the default Korean section title |
| `axis=` | Preserve the section axis such as `workflow`, `visual`, or `system` |
| `tone=` | Preserve an authoring tone hint for later renderer usage |

### `@image`

```md

@end
```

Compiles to:

```md

tag: 필수
::
```

### `@compare`

```md
@compare before=./images/before.webp after=./images/after.webp initial=50
Before/after comparison caption.
@end
```

Compiles to:

```md
::before-after
before: ./images/before.webp
after: ./images/after.webp
initial: 50
caption: Before/after comparison caption.
::
```

### `@video`

```md
@video ./videos/demo.webm poster=./images/poster.webp title="Demo video" muted=true controls=true
Playback caption.
@end
```

Compiles to:

```md
::video-player
src: ./videos/demo.webm
caption: Playback caption.
poster: ./images/poster.webp
title: Demo video
muted: true
controls: true
::
```

### `@gallery` and `@item`

```md
@gallery "Work screens" columns=2 variant=framed
caption: Representative screens and output examples

@item ./images/list.webp "Works list" label=01
@item ./images/detail.webp "Work detail" thumb=./images/detail-thumb.webp label=02
@end
```

Compiles to:

```md
::case-gallery
title: Work screens
caption: Representative screens and output examples
columns: 2
variant: framed
::
- ./images/list.webp | Works list |  | alt=Works list; label=01
- ./images/detail.webp | Work detail | ./images/detail-thumb.webp | alt=Work detail; label=02
::
```

Supported `@item` attributes: `thumb=`, `alt=`, `label=`, and any `media.*` metadata that should be preserved for the gallery parser.

```md
::
```

## 8. Callout box commands

Easy Markdown can wrap the canonical `::markdown-box` directive with short callout commands.

| Easy command | Output |
|---|---|
| `@note` | `::markdown-box type: note` |
| `@tip` | `::markdown-box type: tip` |
| `@warning` | `::markdown-box type: warning` |
| `@danger` | `::markdown-box type: danger` |
| `@ssot` | `::markdown-box type: ssot` |
| `@quote-box` | `::markdown-box type: quote` |
| `@decision-box` | `::markdown-box type: decision` |
| `@box` | `::markdown-box` with explicit type |

### Basic examples

```md
@ssot "SSOT 기준"
상태 귀속 위치를 명시합니다.
@end

@danger "Danger"
실제 실패나 손실 가능성이 있는 상태를 강조합니다.
@end
```

Compiles to:

```md
::markdown-box
type: ssot
title: SSOT 기준
::
상태 귀속 위치를 명시합니다.
::

::markdown-box
type: danger
title: Danger
::
실제 실패나 손실 가능성이 있는 상태를 강조합니다.
::
```

### Collapsible example

```md
@warning "조용한 보정 금지" collapsible=true defaultOpen=false
불명확한 값은 예쁘게 메우지 않고 warning으로 남깁니다.
@end
```

Compiles to:

```md
::markdown-box
type: warning
title: 조용한 보정 금지
collapsible: true
defaultOpen: false
::
불명확한 값은 예쁘게 메우지 않고 warning으로 남깁니다.
::
```

`open=false` is accepted as a short alias and compiles to `defaultOpen: false`.

### Explicit `@box`

```md
@box ssot "기준" tone=blue icon=◆
명시형 박스입니다.
@end
```

Compiles to:

```md
::markdown-box
type: ssot
title: 기준
tone: blue
icon: ◆
::
명시형 박스입니다.
::
```

### `@decision` conflict rule

`@decision` is reserved for case sections.

```md
@decision
판단 서사를 작성합니다.
@end
```

Use `@decision-box` for markdown-box decision callouts.

```md
@decision-box "선택 기준"
박스형 판단 메모를 작성합니다.
@end
```

## 9. Editorial layout commands

Commit 08 adds Easy Markdown wrappers for editorial rhythm blocks. These commands compile to existing VarunTools directives and do not replace canonical `::directive` syntax.

### `@title`

```md
@title "구조와 판단" kicker=VARUNTOOLS subtitle="문제와 해결을 비교합니다." as=h2 align=left
```

Compiles to:

```md
::editorial-title
title: 구조와 판단
kicker: VARUNTOOLS
subtitle: 문제와 해결을 비교합니다.
as: h2
align: left
::
```

The current canonical `editorial-title` renderer supports `as=h1..h4` and `align=left|center`.

### `@columns` and `@col`

```md
@columns 2 gap=md collapse=mobile
@col 문제
문제 설명입니다.

@col 해결
해결 설명입니다.
@end
```

Compiles to:

```md
::editorial-columns
cols: 2
gap: md
collapse: mobile
::
### 문제

문제 설명입니다.

---

### 해결

해결 설명입니다.
::
```

The current canonical `editorial-columns` renderer supports two or three columns.

### Callout boxes

Callout box shortcuts are supported by the callout command family above:

```md
@ssot "SSOT 기준"
Easy Markdown의 원본은 `work.easy.md`이며, `index.md`는 생성물입니다.
@end

@warning "덮어쓰기 주의" collapsible=true open=false
GENERATED marker가 없는 `index.md`는 자동으로 덮어쓰지 않습니다.
@end
```

Use `@box` when you need an explicit markdown-box type:

```md
@box ssot "기준" tone=blue icon=◆
명시형 박스입니다.
@end
```

### `@section-gap` and `@section-break`

```md
@section-gap size=lg

@section-break label="다음 장" tone=quiet
```

Compiles to:

```md
::section-gap
size: lg
::

::section-break
label: 다음 장
tone: quiet
::
```

## 10.1 Generated output and stale checks

Status: implemented in commit 09.

Easy Markdown files are source files. Generated `index.md` files must include the generated marker and source hash.

```md
<!--
GENERATED FROM work.easy.md.
SOURCE HASH: sha256:...
Do not edit this file directly.
Run: npm run easy:compile
-->
```

Use these commands before committing Easy Markdown pages:

```powershell
npm run easy:compile
npm run easy:check
```

Common diagnostics:

| Code | Meaning | Fix |
|---|---|---|
| `EASY007` | Generated `index.md` is stale | Run `npm run easy:compile` |
| `EASY008` | Local asset is missing | Fix the path relative to the easy source file |
| `EASY009` | Related work slug cannot be resolved | Fix or create the related work page |
| `EASY010` | Generated marker is missing | Do not overwrite manual `index.md` without an explicit migration |
| `EASY019` | Source hash does not match | Run `npm run easy:compile` |

## 13. Frontmatter compiler behavior

Commit 04 implements the first active compiler surface. It only compiles metadata/frontmatter commands and leaves body directives for later commits.

Supported commands in this phase:

```txt
# Title
@description
@slug
@public
@hidden
@draft
@cover
@thumb
@work
@role
@stack
@tools
@tags
```

The compiler emits `status: draft` and `visibility: hidden` unless `@public` or `@hidden` is explicitly provided.

Body commands through commit 08 now cover frontmatter, case sections, media blocks, galleries, related works, metric cards, tool-stack blocks, and editorial layout blocks. Store/product wrappers remain reserved for later implementation.

## 15. Demo embed commands

Status: implemented in the iframe demo embed commit.

Easy Markdown can embed isolated project demos with `@demo`. The demo itself runs inside an iframe so the parent Vue app stays protected from demo CSS, canvas, WebGL, and global events.

### Manifest-backed demo

```md
@demo sample-canvas title="Sample Canvas" ratio="16 / 10"
iframe 기반 샘플 데모입니다.
@end
```

Compiles to:

```md
::demo-frame
id: sample-canvas
title: Sample Canvas
ratio: 16 / 10
::
iframe 기반 샘플 데모입니다.
::
```

### Direct src demo

```md
@demo src="demos/sample-canvas/index.html" title="Sample Canvas" fullscreen=false
샘플 데모를 직접 경로로 불러옵니다.
@end
```

Compiles to:

```md
::demo-frame
src: demos/sample-canvas/index.html
title: Sample Canvas
allowFullscreen: false
::
샘플 데모를 직접 경로로 불러옵니다.
::
```

### Rules

| Field | Meaning |
|---|---|
| first positional | demo manifest `id` |
| `id=` | explicit demo manifest id |
| `src=` | direct iframe source |
| `title=` | title override |
| `ratio=` | CSS aspect-ratio value, for example `16 / 10` |
| `status=` | `stable`, `experimental`, or `archived` |
| `fullscreen=` | alias for `allowFullscreen` |
| `allowFullscreen=` | iframe fullscreen permission |
| `sandbox=` | iframe sandbox override |

Demo manifest SSOT:

```txt
src/data/demoManifest.json
```

Static demo entry SSOT:

```txt
public/demos/<demo-id>/index.html
```

Local `src=` values are resolved from `public/`. For GitHub Pages project paths, the runtime component prefixes `import.meta.env.BASE_URL`, so `demos/sample-canvas/index.html` becomes `/varuntools/demos/sample-canvas/index.html` during Pages deployment.

### Auto-resizing iframe demos

Demo iframes can send runtime messages to the parent `DemoFrame` through `public/demos/demo-runtime.js`.
The first implementation intentionally uses `targetOrigin='*'` so GitHub Pages, custom domains, and future CDN-hosted demos do not break during the MVP stage.

```md
@demo sample-canvas autoResize=true minHeight=360 maxHeight=960
샘플 데모입니다.
@end
```

Compiles to:

```md
::demo-frame
id: sample-canvas
autoResize: true
minHeight: 360
maxHeight: 960
::
샘플 데모입니다.
::
```

Supported runtime messages:

| Message | Direction | Meaning |
|---|---|---|
| `VARUN_DEMO_READY` | iframe → parent | demo initialized |
| `VARUN_DEMO_RESIZE` | iframe → parent | demo height changed |
| `VARUN_DEMO_ERROR` | iframe → parent | demo failed internally |

Demo pages can include:

```html
<script src="../demo-runtime.js" defer></script>
```


## Micro Markdown Shorthand Layer

Micro Markdown is the shortest authoring surface in VarunTools. It compiles into Easy Markdown first, then the existing Easy Markdown compiler generates `index.md`.

```txt
work.micro.md
→ work.easy.md
→ index.md
```

Micro Markdown uses uppercase `@COMMAND` shorthands so it does not collide with normal Markdown headings.

### Core example

```md
@H1_바룬의 이야기
@DESC_작업이 계속 쌓여도 무너지지 않도록 만든 개인 출판 엔진.
@SLUG_works/barun-story
@PUBLIC
@COVER=./cover.svg
@THUMB=./thumb.svg
@WORK_type=system,status=published,year=2026,featured=true
@STACK_Vue3, Vite, TypeScript, Markdown

@SUM_
Vue3 + Vite 기반으로 직접 만든 개인 포트폴리오 CMS다.

@SSOT_작성 기준|collapse=true
이 문서의 원본은 work.micro.md이고, work.easy.md와 index.md는 생성물이다.

@BA=./images/before.webp|./images/after.webp|50|작성 흐름 전후 비교

@V=./videos/out.webm|출력 데모 영상|poster=./images/poster.webp,muted=true,controls=true

@DEMO=sample-canvas|Sample Canvas|autoResize=true,minHeight=360,maxHeight=960
```

### Generated Easy Markdown

```md
<!--
GENERATED FROM work.micro.md.
Do not edit this file directly.
Run: npm run micro:compile
-->

# 바룬의 이야기

@description 작업이 계속 쌓여도 무너지지 않도록 만든 개인 출판 엔진.
@slug works/barun-story
@public
@cover ./cover.svg
@thumb ./thumb.svg

@work type=system status=published year=2026 featured=true
@stack Vue3, Vite, TypeScript, Markdown

@summary
Vue3 + Vite 기반으로 직접 만든 개인 포트폴리오 CMS다.
@end

@ssot "작성 기준" collapsible=true defaultOpen=true
이 문서의 원본은 work.micro.md이고, work.easy.md와 index.md는 생성물이다.
@end

@compare before=./images/before.webp after=./images/after.webp initial=50
작성 흐름 전후 비교
@end

@video ./videos/out.webm title="출력 데모 영상" poster=./images/poster.webp muted=true controls=true
@end

@demo sample-canvas title="Sample Canvas" autoResize=true minHeight=360 maxHeight=960
@end
```

### Supported shorthands

| Micro syntax | Easy Markdown output |
|---|---|
| `@H1_Title` | `# Title` |

### Commands

```powershell
npm run micro:compile
npm run micro:check
npm run smoke:micro
```

`micro:compile` writes the intermediate Easy Markdown file and then runs the Easy Markdown compiler, so generated `work.easy.md` and `index.md` stay in sync.

### Collision rule

Normal Markdown headings remain valid:

```md
# Normal Markdown heading
```

Only uppercase `@COMMAND` lines are treated as Micro Markdown commands.

## Micro Markdown Scaffolding

Micro Markdown pages can be scaffolded from the CLI.

```powershell
npm run new:work:micro -- barun-story --title "바룬의 이야기"
npm run new:page:micro -- guide markdown-guide --title "Markdown Guide"
```

Generated Works folders contain:

```txt
work.micro.md
work.easy.md
index.md
README.md
cover.svg
thumb.svg
images/.gitkeep
videos/.gitkeep
```

Generated regular page folders contain:

```txt
page.micro.md
page.easy.md
index.md
README.md
cover.svg
images/.gitkeep
videos/.gitkeep
```

SSOT rule:

```txt
work.micro.md / page.micro.md = source
work.easy.md / page.easy.md = generated intermediate
index.md = generated final output
```

## Compiler Order and Build Integration

VarunTools authoring sources compile in this order:

```txt
Micro Markdown
→ Easy Markdown
→ Canonical Markdown
→ Vue render
```

Commands:

```powershell
npm run micro:compile
npm run easy:compile
npm run content:compile
npm run content:check
npm run build
```

`npm run build` automatically runs `prebuild`, so Micro Markdown and Easy Markdown sources are compiled and checked before Vite builds the site.

Pipeline scripts:

```json
{
  "content:compile": "npm run micro:compile && npm run easy:compile",
  "content:check": "npm run micro:check && npm run easy:check",
  "prebuild": "npm run content:compile && npm run content:check"
}
```
