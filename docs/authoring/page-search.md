# Page Search

Commit 105 adds a site-wide local page search index.

## SSOT

The search index is generated from routed content and authoring documentation metadata:

- `frontmatter`
- `frontmatter.work`
- Markdown body excerpts
- SEO indexability policy

Generated output:

```txt
src/content/generated/page-search-index.json
```

Do not edit this file manually. Regenerate it with:

```bash
npm run build:page-search
```

## Page search vs portfolio search

`/works` search is a portfolio collection filter.

```txt
/works?q=vue
```

`/search` is site-wide page search.

```txt
/search?q=csv
/search?q=포트폴리오
```

These two search states are intentionally separate.

## Included sources

- routed content pages under `src/content/pages/**/index.md`
- portfolio work pages
- product/store pages
- policy/general pages
- authoring docs under `docs/authoring/**/*.md`

## Excluded sources

- `src/content/templates/**`
- `src/content/generated/**`
- fixtures
- draft/private/hidden pages
- noindex pages
- admin/API/runtime-only surfaces

## Search fields

The generated index includes searchable text from:

- title
- description
- summary
- body excerpt
- slug/href
- section
- tags
- keywords
- `work.tags`
- `work.stack`
- `work.role`
- `work.tools`
- product category data

## Scoring

The runtime search uses a small local score, without external search dependencies:

```txt
title match       +50
slug/href match   +30
tag match         +25
keyword match     +25
description match +15
body text match   +5
```

Results sort by score, then type priority, then title.

## URL query sync

The search page owns its own query state:

```txt
/search?q=csv
```

Query URLs are not added to `sitemap.xml`. Only the base `/search` route can be indexable.

## Commands

```bash
npm run build:page-search
npm run smoke:page-search
```

## Release checklist

Before publishing:

```txt
[ ] npm run build:page-search
[ ] npm run smoke:page-search
[ ] /search route exists
[ ] /search?q=... works locally
[ ] generated/templates are absent from the index
[ ] draft/private/hidden pages are absent from the index
[ ] sitemap contains /search but not /search?q=...
```

## Publish gate relation

`npm run check:publish` reads `page-search-index.json` to verify that public works are discoverable through site-wide page search.
