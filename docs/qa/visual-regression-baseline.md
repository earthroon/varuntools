# Visual Regression Baseline

Commit 62 adds the baseline/diff layer on top of the Commit 61 Playwright screenshot harness.

```txt
Commit 60 = manual QA criteria
Commit 61 = screenshot camera
Commit 62 = baseline comparer
```

## Commands

```bash
npm run qa:screenshots
npm run qa:baseline
npm run qa:diff
npm run qa:diff:strict
npm run qa:diff:summary
npm run smoke:visual-diff
```

## Directory Contract

```txt
artifacts/qa/screenshots/current/   generated by qa:screenshots
artifacts/qa/screenshots/baseline/  promoted by qa:baseline
artifacts/qa/screenshots/diff/      generated by qa:diff
artifacts/qa/screenshots/report.json
artifacts/qa/screenshots/report.md
```

`current`, `diff`, `tmp`, and generated reports are ignored. The baseline directory itself is not globally ignored so an accepted baseline can be committed later if the project chooses to do that.

## Baseline Promotion

`npm run qa:baseline` copies the current screenshots into the baseline directory and writes `artifacts/qa/screenshots/baseline/manifest.json`.

The baseline is never updated by qa:diff. This is the important guardrail. A regression must not silently become the new reference.

## Diff Behavior

`npm run qa:diff` compares each planned screenshot pair.

```txt
missing current = error
missing baseline = warning
dimension mismatch = error
small mismatch = warning
large mismatch = error
```

`npm run qa:diff:strict` treats warnings as failures. Use strict mode for release freeze or after a baseline has been intentionally promoted.

## Thresholds

```bash
QA_DIFF_CHANNEL_THRESHOLD=12
QA_DIFF_WARNING_THRESHOLD=0.001
QA_DIFF_ERROR_THRESHOLD=0.01
```

## Report Files

`qa:diff` writes `artifacts/qa/screenshots/report.json` and `artifacts/qa/screenshots/report.md`.

## Non-goals

```txt
No automatic baseline refresh during diff.
No Playwright toHaveScreenshot assertions yet.
No CI release blocking by visual diff until the baseline policy is chosen.
No image snapshot files are generated during check:launch.
```
