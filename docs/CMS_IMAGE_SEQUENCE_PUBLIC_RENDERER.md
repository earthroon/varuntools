## Scope

VT-CMS-02 mounts the `image-sequence` custom element generated by VT-CMS-01.
It reads the JSON payload from `template[data-image-sequence-items]`, resolves each `src` through the existing public content asset resolver, and renders a crop-strip image sequence.

## SSOT

Input SSOT:

```txt
<image-sequence>
  <template data-image-sequence-items>JSON</template>
</image-sequence>
```

Runtime SSOT:

```txt
ImageSequenceItem[] from template JSON order
```

The public repo does not reorder items. CMS authoring order is preserved.

## Explicit non-goals

- No D1 lookup
- No R2 list lookup
- No assetId reverse lookup
- No image optimizer
- No srcset
- No sizes
- No EWA or WebGPU image processing
- No transform URL generation
