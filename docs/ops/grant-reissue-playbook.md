# Grant Reissue Playbook

Grant reissue requires manual review. Reissue candidates include R2_OBJECT_MISSING, legitimate expiry support request, download limit exception, wrong deliverable set, or paid order with failed claim guidance. Reissue forbidden candidates include unverified payment, refunded order, fraud-suspected revoke reason, unclear productSlug, or identity/order mismatch. Do not create a new grant in Commit 78. Manual review required.


Commit 80 dry-run planning must preserve variant/bundle entitlement scope and must not silently expand rights.
